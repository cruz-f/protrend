best practices

ProTReND provides open programmatic access to the Transcriptional Regulatory Network (TRN) database through a RESTful web API. All data can be visualized by navigating through the several biological entities available at the API Index.

Biological entities:

  • Effector
  • Evidence
  • Gene
  • Operon
  • Organism
  • Pathway
  • Publication
  • Regulator
  • RegulatoryFamily
  • RegulatoryInteraction
  • Source
  • TFBS

In addition, ProTReND REST API provides data access through pre-built datasets in the following formats:

  • TRNs
  • Organisms Binding Sites
  • Regulators Binding Sites

The following methods are available to all users over all end-points:

  • GET
  • HEAD
  • OPTIONS

The following methods are available to registered users (upon request) over all end-points:

  • GET
  • POST
  • PUT
  • PATCH
  • DELETE
  • HEAD
  • OPTIONS

All requests submitted to the REST API must be performed in JSON format. Responses provided by the REST API will also be provided in a JSON format by default. ProTReND REST API also supports data access in the following file formats:

  • JSON
  • CSV
  • EXCEL
  • BROWSABLE HTML API
  • FASTA (Binding Sites datasets)

For more details about each API end-point consult the swagger and redoc documentation

swagger redoc

REQUEST POLICY:

  • ANONYMOUS USERS PERFORMING MORE THAN 3 REQUESTS PER SECOND WILL BE BANNED!
  • REGISTERED USERS PERFORMING MORE THAN 5 REQUESTS PER SECOND WILL BE BANNED!
  • For large requests, please submit GET request to the list end-points, such as api/genes/, api/regulators/, etc
